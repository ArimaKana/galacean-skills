# Shader 调试清单（速查版）

目标：2-5 分钟内定位 80% 着色器问题。按顺序检查，不要跳步。

## 0. 先确认现象
- 是“完全不显示”还是“显示但效果不对”。
- 只在某台设备异常，还是全平台异常。

## 1. 完全不显示（优先级最高）
1. 组件是否启用：`renderer.enabled === true`。
2. Sprite/网格是否存在：`sprite` 或 mesh 不是空。
3. 材质是否挂上去：`renderer.setMaterial(material)` 是否执行。
4. 相机是否能看到：位置、层级、裁剪面是否正确。
5. 透明对象是否开启：`material.isTransparent = true`。
6. alpha 是否被乘为 0：先临时输出 `gl_FragColor = vec4(1,0,0,1)`。

## 2. 参数不生效
1. uniform 名称是否完全一致（大小写敏感）。
2. 是否更新了“当前正在渲染”的同一 material 实例。
3. 每帧是否被其他逻辑覆盖（例如重建材质后丢参数）。
4. 数值范围是否合理：强度先夹到 [0, 1]。

## 3. 方向/位置不对
1. UV 方向是否需要翻转：`uvY = 1.0 - v_uv.y`。
2. world/local 坐标是否混用。
3. 动画时间单位是否统一为秒。

## 4. 闪烁、抖动、跳变
1. 时间是否累计稳定（避免每帧重置为 0）。
2. 临界值比较改为 `smoothstep` 软边过渡。
3. 高频参数降频更新（每 2 帧一次）验证是否稳定。

## 5. 性能异常（发热/掉帧）
1. Shader 是否重复创建：用 `Shader.find + Shader.create` 缓存。
2. 是否每帧 `new BaseMaterial()`：改为复用 + 更新 `shaderData`。
3. 片元分支是否过多：减少复杂 `if` 链。
4. 纹理采样是否过重：先用纯色分支定位瓶颈。

## 6. 快速回归法（推荐）
1. 输出常量红色，验证渲染链路是否通。
2. 只保留 `material_BaseColor`，验证材质绑定是否通。
3. 再逐个加回时间、波形、遮罩逻辑，定位首个出错点。

## 7. 团队统一约定
- Shader 名统一加版本后缀：`*-v1`、`*-v2`。
- uniform 统一 `material_` 前缀。
- 调试结束前移除临时代码（纯色输出、日志轰炸）。
